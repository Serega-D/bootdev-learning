# Gator 🐊 - RSS Feed Aggregator CLI

**Gator** — это консольный агрегатор RSS-лент, написанный на языке Go. Программа позволяет регистрировать пользователей, подписываться на различные RSS-источники, автоматически собирать новые посты в фоновом режиме и просматривать их в удобном формате прямо в терминале.



## Требования

Для запуска проекта вам понадобятся:
* **Go** версии 1.25.7 или выше.
* **PostgreSQL** (убедитесь, что сервер запущен).
* **Goose** — инструмент для управления миграциями базы данных.

## Установка

1. **Клонируйте репозиторий**:
   ```bash
   git clone [https://github.com/Serega-D/bootdev-learning.git](https://github.com/Serega-D/bootdev-learning.git)
   cd bootdev-learning/rss_gator

   Соберите бинарный файл:
    Bash

    go build -o gator

    Настройте базу данных:
    Создайте новую БД в Postgres (например, gator) и примените миграции:
    Bash

    goose -dir sql/schema postgres "postgres://your_user:your_password@localhost:5432/gator" up

    Конфигурация:
    При первом запуске команды register или login, программа создаст файл .gatorconfig.json в вашем домашнем каталоге. Убедитесь, что параметр db_url содержит корректную строку подключения к вашей БД.

Основные команды
Управление пользователями

    register <name> — создать нового пользователя и войти под ним.

    login <name> — переключиться на существующего пользователя.

    users — список всех зарегистрированных пользователей.

Управление лентами

    addfeed <name> <url> — добавить новую ленту и автоматически подписаться на неё.

    feeds — показать все ленты в системе.

    follow <url> — подписаться на существующую ленту.

    unfollow <url> — отписаться от ленты.

    following — список ваших активных подписок.

Сбор и чтение данных

    agg <interval> — запустить воркер для сбора постов (например, ./gator agg 1m или ./gator agg 30s). Программа будет опрашивать ленты в фоновом режиме.

    browse [limit] — показать последние посты из ваших подписок. По умолчанию выводит 2 поста. Пример: ./gator browse 10.

Технологии

    SQLC — генерация типобезопасного кода Go из SQL-запросов.

    Goose — версионирование схемы БД.

    UUID — использование уникальных идентификаторов для всех записей.

    Concurrency — использование тикеров и контекстов для стабильной работы агрегатора.
